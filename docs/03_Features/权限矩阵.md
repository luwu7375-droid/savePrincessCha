## 一、核心原则

  

三条铁律：

  

1. **user = 最高控制权（现实锚点）**

2. **cha = 可行动体，但不能定义事实**

3. **system = 规则执行者，不参与内容创作**

4. **external = 黑盒，只能调用不能篡改**

  

---

  

# 二、权限矩阵总表

  

## 1｜内容生成权限

  

|行为|user|cha|system|external|

|---|---|---|---|---|

|发帖子（朋友圈/论坛）|✔|✔（受限）|✖|✖|

|生成游戏对话|✔|✔|✔（规则驱动）|✖|

|写日记|✔|✔|✖|✖|

|生成叙事总结|✔|❌|✖|✖|

|生成“我们发生了什么”|✔|❌|✖|✖|

  

👉 关键点：

  

- cha **不能生成事实叙事**

- system **不能写内容**

- user 可以“创造叙事”（因为这是现实锚点）

## 3｜写入权限（最关键）

  

|数据类型|user|cha|system|external|

|---|---|---|---|---|

|events（主时间线）|✔|✔（通过系统）|✔|✔（通过API）|

|is_favorite（瞬间）|✔|❌|✔|✖|

|ai_tags|❌|✔（自动）|✔|✖|

|system_event|✔|✔（触发）|✔|✔|

|记忆沉淀|✔（间接）|✔（建议）|✔（执行）|✖|

  

👉 关键点：

  

- cha **不能直接写收藏权重**

- cha 只能“建议”，不能“决定重要性”

  

---

  

## 4｜事实定义权限（最敏感）

  

|行为|user|cha|system|

|---|---|---|---|

|定义“发生过什么”|✔|❌|✔（记录，不解释）|

|修改历史事件|✔（允许编辑）|❌|❌|

|删除事件|✔|❌|✔（soft delete）|

|编造历史|✔（模拟允许）|❌|❌|

  

👉 核心：

  

> “事实层只有 system 写入，cha 永远不能改事实”

|模拟能力|user|cha|system|

|---|---|---|---|

# 三、系统级三条边界（必须写进代码注释）

  

## 1｜事实边界（Fact Boundary）

  

```

只有 system 写事实层cha 不能定义历史user 可以编辑但需显式操作

```

  

---

  

## 2｜叙事边界（Narrative Boundary）

  

```

cha 可以表达但不能生成“总结关系/我们如何变化”

```

  

---

  

## 3｜权重边界（Weight Boundary）

  

```

is_favorite 只能 user 触发ai_tags 只能 system/AI标注cha 不能参与权重决策

```

  

---

  

# 四、最关键的一句话总结（帮你定架构心智）

  

👉 Playground 的本质权限结构是：

  

> user 决定“什么重要”  

> system 决定“什么真实”  

> cha 负责“如何参与”  

> external 提供“世界素材”
