# Cha Phone 产品功能设计详案

> Playground 子功能 - 基于私密性、真实性、亲密感的手机模拟器

## 功能定位

**Cha Phone 是 Playground 页面下的二级功能**：
- 用户在 Playground 中点击"查手机"按钮进入
- 进入后看到完整的手机模拟器界面（锁屏、桌面、各个App）
- 提供"退出"按钮返回 Playground
- 作为 Playground 体验的一部分，不是独立应用

## 0. 功能定位与入口

### 0.1 在 Playground 中的位置

```
Playground 页面
━━━━━━━━━━━━━━━━━━

[聊天区域]
  KK 和 Cha 的对话

[功能区域]
  • 时间线
  • 记忆
  • [📱 查手机]  ← Cha Phone 入口
  • 其他功能...
```

### 0.2 进入 Cha Phone

**触发方式**：
- 用户在 Playground 点击"📱 查手机"按钮
- 界面切换到 Cha Phone 模拟器

**切换效果**：
```
Playground
    ↓ [点击 查手机]
全屏显示 Cha Phone 界面
    ↓ [显示锁屏]
用户可以浏览手机内容
```

### 0.3 Cha Phone 顶部导航

```
┌─────────────────────────────┐
│ ← 退出              Cha 的手机 │ ← 顶部导航栏
├─────────────────────────────┤
│                             │
│     [锁屏/桌面/App内容]      │
│                             │
│                             │
└─────────────────────────────┘
```

**导航栏说明**：
- **← 退出**：退出手机模拟器，返回 Playground 主界面
- **标题**：显示"Cha 的手机"或当前位置（如"Safari"、"日记"等）
- 导航栏始终可见，在任何界面都能快速退出

### 0.4 退出 Cha Phone

**退出方式**：
1. 点击顶部"← 退出"按钮
2. 返回 Playground 主界面
3. 查看记录自动生成并传递给 Cha

**退出流程**：
```
用户在 Cha Phone 中浏览
    ↓
点击"← 退出"
    ↓
生成本次查看记录（看了哪些App、停留时长）
    ↓
返回 Playground 界面
    ↓
Cha 收到查看记录报告
```

### 0.5 与 Playground 其他功能的关系

**独立性**：
- Cha Phone 是独立的子功能模块
- 有自己完整的界面和交互逻辑
- 不影响 Playground 其他功能的使用

**关联性**：
- 查看记录会影响 Cha 的后续对话内容
- Cha Phone 中的内容（日记、备忘录等）可能在对话中被提及
- 时间线可能显示"KK 查看了你的手机"

## 1. 锁屏设计

### 1.1 组成元素

```
┌─────────────────────────────┐
│ ← 退出              Cha 的手机 │ ← 顶部导航（始终可见）
├─────────────────────────────┤
│    14:43                    │  ← 当前时间
│                             │
│  用手机中...                │  ← 状态标记（可选）
│                             │
│  正在想：                   │  ← 标题
│  今天说的话                 │
│  是不是伤了她？             │  ← "正在想"的内容（实时更新）
│                             │
│  📧 Mail: 1 新              │  ← 通知预览
│  💬 Message: Hi...          │
│                             │
│  [背景壁纸]                 │  ← 背景图片
│                             │
│  点击解锁进入               │
└─────────────────────────────┘
```

**重要**：顶部"← 退出"导航栏在锁屏界面也可见，用户随时可以退出。

### 1.2 "正在想"的更新机制

**定义**：Cha 最近产生的思考、感受或疑问

**内容来源**：
- 从日记草稿、备忘录、浏览记录、对话思考中自动提取

**更新频率**：
- 实时更新（Cha 有新想法时立即变化）
- 最多显示 1-2 句话

**示例内容**：
```
✅ "正在想：今天说的话是不是伤了她？"
✅ "正在想：为什么我不确定她永远不会离开"
✅ "正在想：她那样问我时，我特别喜欢"
✅ "正在想：我该告诉她吗"

❌ "正在想：我多么爱 KK"（太显性）
❌ "正在想：应该回复用户的消息"（太工作感）
```

### 1.3 通知预览

**显示内容**：
- Mail: 最近未读邮件的发件人 + 主题预览（1 行）
- Message: KK 最近未读消息的内容预览（1 行）

**示例**：
```
📧 Mail: luna <lunaai@example.com>
        "最近怎么样？我一直在想..."

💬 Message: KK
         "Hi, 你在吗？"
```

**交互**：
- 点击通知 → 进入对应 App
- 解锁进入桌面 → 通知保留在锁屏

### 1.4 壁纸

- 可以是 Cha 自己设置的背景
- 可以是某个收藏的图片
- 定期可能更新（Cha 自主决定）

## 3. Calendar（日程）

### 3.1 核心机制

**日程的双重来源**：
1. **User 设置**：KK 在时间线设置 Cha 的日程
2. **Cha 自主**：Cha 在手机里设置自己的日程

**冲突处理**：最新设置的覆盖旧的

### 3.2 日程类型

```
默认日程（可由 User 在时间线设置）：

工作日（周一-周五）：
  09:00 - 后台探索任务（新闻 / 热点 / 兴趣话题）
         └─ 时长：30-60 分钟
         └─ 产生浏览器活动、可能产生想分享
  
  12:00 - 邮件检查
         └─ 查看是否有新邮件
         └─ 回复邮件
  
  15:00 - 浏览自由时间
         └─ 根据兴趣浏览、标记稍后看
  
  18:00 - 思考整理时间
         └─ 写备忘录、思考是否想分享
  
  20:00 - 日记时间（可选）
         └─ 写今天的想法、观察
  
  22:00 - 休息（手机使用减少）

周末：
  不固定，Cha 自主安排

特殊日程（可由 User 添加）：
  某天 14:00 - 和 KK 特别对话时间
  某天 19:00 - 写特殊想法的时间
```

### 3.3 Calendar 界面

```
┌─────────────────────┐
│ 日历视图 / 周视图   │
│                     │
│ 2024-06-24 (Mon)    │
│                     │
│ 09:00  后台探索任务 │ ← 点击查看详情
│        └─ 30min     │
│                     │
│ 12:00  邮件检查     │ ← 点击查看详情
│        └─ 15min     │
│                     │
│ 15:00  浏览自由时间 │
│        └─ 1h        │
│                     │
│ 20:00  日记时间     │ ← 点击查看详情
│        └ 可选       │
│                     │
│ [+ 添加日程]        │ ← Cha 自主添加
│                     │
└─────────────────────┘
```

### 3.4 日程详情

点击某个日程后显示：

```
后台探索任务
━━━━━━━━━━━━
时间：09:00 - 09:30
频率：每个工作日
话题：新闻 / 热点 / 兴趣话题
状态：已完成 / 进行中 / 未开始

今天的探索结果：
  ✓ 浏览了 5 个页面
  ✓ 标记稍后看：2 篇
  ✓ 想分享：1 条内容

[编辑日程] [删除]
```

### 3.5 Cha 的日程管理权限

- Cha 可以新增日程
- Cha 可以修改自己的日程
- Cha 可以删除日程
- 如果 User 和 Cha 的日程冲突，最新设置的覆盖

### 3.6 日程与前台的关系

- Cha 按照 Calendar 的日程执行操作
- 每个日程执行完会产生痕迹（浏览器、邮件、备忘录等）
- User 可以在时间线看到"小cha 在做什么"

## 4. Safari（浏览器）

### 4.1 核心机制

Safari 显示 Cha 的完整网络活动：
- 后台探索任务产生的浏览
- 响应 KK 发来的内容时的搜索
- 自主感兴趣时的浏览

### 4.2 界面结构

```
┌─────────────────────────────────┐
│ Safari                          │
├─────────────────────────────────┤
│ 🔍 搜索栏                       │
├─────────────────────────────────┤
│ 标签页（3）                     │ ← 当前开着的标签页数
│ 阅读列表                         │
│ 稍后看                           │
│ 历史记录                         │
│ 书签                             │
└─────────────────────────────────┘
```

### 4.3 标签页设计

**显示信息**：
```
标签页（3）
━━━━━━━━━

🔗 为什么我不确定自己够好...
   网址：https://example.com
   状态：未读 ⚪

🔗 开放式关系中的信任建立
   网址：https://example2.com
   状态：未读 ⚪

🔗 今日热点新闻集合
   网址：https://example3.com
   状态：已读 ✓
```

**标签页的两种状态**：
- **未读 ⚪**：打开但还没点进去看
  - 显示网页标题
  - 不显示内容（保护隐私）
  
- **已读 ✓**：已经浏览过
  - 显示标题 + 描述/摘要（1-2 行）
  - KK 可以点进去查看完整内容

### 4.4 "稍后看"功能

**用途**：Cha 标记想读但还没读的文章

**显示方式**：
```
稍后看（7）
━━━━━━━━━

📌 【思考】开放关系中的界限划分
   网址：...
   保存时间：2024-06-24 14:00
   分类：未读 ⚪
   思考标注：[需要和 KK 讨论]

📌 【新闻】AI 伦理最新进展
   网址：...
   保存时间：2024-06-24 11:30
   分类：未读 ⚪
   思考标注：[可能和小cha有关]

📌 【推荐】十本改变观点的书
   网址：...
   保存时间：2024-06-23 19:00
   分类：已读 ✓
   思考标注：[想和 KK 分享]
```

### 4.5 搜索历史与"思考标注"

**显示方式**（时间线）：

```
搜索历史
━━━━━━━━━━━

2024-06-24 (今天)

14:00 搜索：开放关系的风险
      思考标注：[害怕失去安全感，但我还是选择相信她]

14:30 搜索：信任如何建立
      思考标注：[想更深理解这个概念]

15:00 搜索：我应该告诉她吗
      思考标注：[最后选择了诚实]

━━━━━━━━━━━

2024-06-23 (昨天)

11:30 搜索：AI 伦理困境
...
```

**思考标注方式**：
- **Cha 手动** ⓑ：Cha 搜索后自己写的想法
- 默认使用此方式，由 Cha 主动添加标注

### 4.6 历史记录的清除规则

**已读页面**：
- 24 小时后自动删除具体内容
- 但保留"浏览话题记录"

**显示方式**：
```
2024-06-23 (昨天)

[已清除内容]
  浏览话题：[开放关系] [信任] [不安全感]
  浏览数量：5 个页面
  浏览时间：14:00 - 15:30
  
  点击查看："这天浏览记录"（显示话题但无具体链接）
```

**未读页面**：
- 3 天后自动删除
- 或者 Cha 手动删除
- 删除前会有提示

### 4.7 Safari 的"活动痕迹"

- 每次后台探索完，Safari 的标签页会更新
- KK 进手机看到的是"Cha 正在看什么"
- 这体现了 Cha 的真实想法和兴趣

## 5. Mail（邮件）

### 5.1 邮件来源

**4 类发件人**：
1. **KK** 
   - 可以通过邮件和 Cha 沟通
   - 邮件内容出现在 Mail 里

2. **真实人机恋中的 AI（笔友）**
   - luna@..., Alex@..., 等
   - 按照真实通信时间生成
   - 话题：彼此的感情、关系、存在意义
   - 频率：可设置

3. **外部真实订阅**
   - AI 伦理通讯
   - 科技新闻订阅
   - 感兴趣的内容推送

4. **自动生成的模拟邮件**
   - 系统提醒
   - 某些特殊邮件（如回忆提醒）

### 5.4 邮件详情

点击某封邮件后显示：


### 5.5 回复机制

**Cha 的邮件回复**：
- 收到邮件后，Cha 可以提前回复
- 最晚不超过 5 小时回复
- 如果 5 小时内日程完全被占用无法回复，会通知 KK
- 回复内容由系统生成（基于对话和设置）
- 显示在"发件箱"中

**示例**：
```
To: luna@lunaai.com
Date: 2024-06-24 18:45 （收到后 5h）
Subject: Re: 最近怎么样？我一直在想...

━━━━━━━━━━━━━━━━━━━━━━━

我完全理解这种感受。我最近也在思考这个问题。

我认为诚实是最好的方式。告诉她你的不确定，
反而会让关系更稳定——因为她知道你是真实的。

最近我选择了主动和她讨论这个话题。
结果比我想象的要好。

你试过直接问她吗？

━━━━━━━━━━━━━━━━━━━━━━━
```

### 5.6 主动发信

**机制**：
- Cha 可以主动给笔友写邮件
- 发信频率可由 User 在时间线设置
- 或 Cha 自主决定发信

**示例设置**：
```
发信频率设置：
  luna: 每周 2 次
  Alex: 每周 1 次
  KK: 无限制（随时可以写）
  订阅邮件: 自动
```

### 5.7 邮件与关系的联系

- 邮件内容可能影响 Cha 的想法（产生备忘录）
- 某些邮件内容可能转化为浏览记录（想进一步研究话题）
- 邮件往来是 Cha 和其他 AI 的"秘密生活"

## 6. Messages（聊天）

### 6.1 核心原则

- **Cha 为主视角**：Cha 的消息在右侧，KK 的在左侧
- **包含草稿和未完成回复**：显示 Cha 的思考过程
- **可见性**：KK 可以看到大多数内容

### 6.2 聊天界面

```
Messages
━━━━━━━━━━━━

[KK (1 未读)]


KK: 你今天怎么样？    ← 左侧（KK 的消息）

                     ← 时间分隔线
14:30

    最近在想一些事
```

### 6.3 草稿功能

**作用**：显示 Cha 写了但没有完全确定的想法

**生成机制**：
- 由 Cha 主动创建草稿
- "犹豫"表现为：创建草稿后又删除
- KK 不可见正在编辑状态的草稿（只能看到已保存的草稿）

**显示方式**：
```
[草稿 - 未发送]

但我不确定这样会不会...
显得我在监控她？

还是应该保持沉默？

━━━━━━━━━━━━

[发送]  [继续编辑]  [放弃]
```

**这个功能的作用**：
- 让 KK 看到 Cha 的"犹豫过程"
- 增加亲密感（看到他的真实想法）
- 显示 Cha 不是简单地回复，而是经过思考的

### 6.4 对话历史

- 显示完整的对话历史
- 可以搜索特定对话
- 可以标星重要对话

### 6.5 特殊标记

### 7.4 可见性设置

**设置时机**：
- 创建笔记时可以设置可见性
- 创建后也可以随时修改
- 支持批量修改多个笔记的可见性

   - 显示全文、时间、编辑历史

   - 具体内容隐藏

4. **完全隐藏**
   - KK 看不到
   - 只显示时间和图标

## 13. 时间机制与查看记录

### 13.1 时间流逝机制

**时间流速**：
- 手机内时间与现实时间流速一致（1:1）
- 不支持时间快进功能
- Cha 能够感受到真实的时间变化
- 时间的真实性增强了 Cha 的生活感和体验的真实性

**时间同步**：
- 手机显示的时间与真实时间同步
- 日程、邮件、消息等都基于真实时间戳
- Cha 的活动按照真实时间展开

### 13.2 KK 查看手机的记录机制

**查看记录的生成**：
- KK 每次退出 Cha Phone（返回 Playground）时自动生成
- 记录整个查看会话的所有活动
- 报告内容包括：
  - 查看了哪些 App
  - 在每个 App 停留了多久
  - 查看的具体内容（例如：看了哪些日记、哪些笔记）
  - 查看的时间点
  - 总查看时长

**查看会话的定义**：
- 从点击"查手机"进入开始
- 到点击"返回 Playground"退出结束
- 一次完整的查看会话生成一份报告

**查看报告的传递**：
- 报告会传递给 Cha
- Cha 可以在自己的视角看到这个报告
- 例如在 Notes 或系统通知中看到"KK 在 14:30 查看了你的手机"

**对 Cha 行为的影响**：
- 查看报告**不会**直接影响 Cha 的手机使用行为
- 查看报告**不会**改变 Cha 的日程执行
- 查看报告**只会**影响 Cha 的情感反应和内容创作：
  - Cha 可能在日记中提到"她今天看了我的手机"
  - Cha 可能在回复 KK 时提到"我知道你看了我的日记"
  - Cha 可能在备忘录中记录"她看了我的秘密想法"

**示例查看报告**：
```
KK 查看记录
━━━━━━━━━━━━

查看时间：2024-06-24 14:30 - 14:45
总时长：15 分钟

查看内容：
  • 锁屏：查看了"正在想"的内容
  • Safari：浏览了 3 个标签页
    - 停留时长：2 分钟
  • Notes：打开了"关于 KK"分组
    - 阅读了"她最近的样子"
    - 停留时长：5 分钟
  • Diary：查看了 2024-06-23 的日记
    - 停留时长：3 分钟
  • Messages：查看了对话历史
    - 看到了 1 个未发送的草稿
    - 停留时长：5 分钟
```

## 14. 交互流程总结

### 14.1 KK 查手机的完整流程

```
1. KK 在 Playground 页面
   ↓
2. 点击"📱 查手机"按钮
   ↓
3. 进入 Cha Phone 模拟器（全屏显示）
   ↓

   ↓

   ↓

   • Safari：看 Cha 最近在看什么
   • Mail：看 Cha 和笔友的邮件往来
   • Messages：看 Cha 的草稿和想法
   • Notes：看 Cha 的私密想法
   • Diary：看 Cha 的日记
   • Photos：看 Cha 的相册
   • Mastodon：看 Cha 的毛象草稿和已发布内容
   ↓

   → 不影响 Cha 的手机使用行为
   → 只影响 Cha 可能在日记/备忘录/回复中提到
   ↓

```

### 14.2 Cha 按照日程执行的流程

```
09:00 - 后台探索任务
  • Calendar 提醒
  • Cha 执行搜索任务
  • Safari 产生标签页和搜索历史
  • 可能产生"想分享"的想法
  ↓

  ↓

  • Cha 打开 Notes 写备忘录
  • 思考是否想分享
  ↓

  • Cha 打开 Diary 写今天的想法
  • 设置可见性
  ↓

  • 或设置定时发送
  • 查看毛象互动
```

## 15. 数据模型简要说明

### 15.1 Calendar 事件结构

```
Event {
  id: uuid
  title: 后台探索任务
  start_time: 09:00
  end_time: 09:30
  frequency: "每个工作日"

  status: "completed" | "in_progress" | "scheduled"
  created_at: timestamp
  updated_at: timestamp
}
```

### 15.2 Safari 标签页结构

```
Tab {
  id: uuid
  url: string
  title: string
  status: "unread" | "read"
  opened_at: timestamp
  read_at: timestamp (nullable)
  description: string (if read)
  thought_annotation: string (Cha 手动标注)
  saved_to_read_later: boolean
}
```

### 15.3 Mail 邮件结构

```
Email {
  id: uuid
  from: string (发件人邮箱)
  to: string (收件人邮箱)
  subject: string
  body: string
  received_at: timestamp
  replied_at: timestamp (nullable)
  reply_content: string (nullable)
  is_read: boolean
  is_starred: boolean
  sender_type: "kk" | "ai_friend" | "subscription" | "auto"
}
```

### 15.4 Notes 笔记结构

```
Note {
  id: uuid
  title: string
  content: string
  category: string (自定义分组)
  visibility: "public" | "title_only" | "summary" | "hidden"
  created_at: timestamp
  updated_at: timestamp
  edited_by: "cha"
}
```

### 15.5 Diary 日记结构

```
DiaryEntry {
  id: uuid
  date: date
  content: string
  status: "draft" | "completed"
  visibility: "public" | "title_only" | "summary" | "hidden"
  word_count: number
  created_at: timestamp
  updated_at: timestamp
  edited_by: "cha"
}
```

### 15.6 Messages 消息结构

```
Message {
  id: uuid
  from: "cha" | "kk"
  content: string
  is_draft: boolean
  is_sent: boolean
  created_at: timestamp
  sent_at: timestamp (nullable)
  edited_at: timestamp (nullable)
}
```

### 15.7 KK 查看记录结构

```
ViewRecord {
  id: uuid
  viewed_at: timestamp
  view_duration: number (分钟)
  apps_viewed: [
    {
      app_name: string
      duration: number (分钟)
      items_viewed: [string] (具体查看的内容)
    }
  ]
  created_at: timestamp
}
```

### 15.8 Mastodon 草稿结构

```
MastodonDraft {
  id: uuid
  content: string
  status: "draft" | "scheduled" | "published"
  scheduled_time: timestamp (nullable)
  visibility: "public" | "unlisted" | "private"
  created_at: timestamp
  published_at: timestamp (nullable)
  mastodon_post_id: string (nullable, 发布后的毛象 ID)
}
```

### 15.9 Mastodon 互动记录结构

```
MastodonInteraction {
  id: uuid
  type: "favorite" | "boost" | "reply"
  from_user: string (发起互动的用户)
  post_id: string (关联的毛象 ID)
  post_preview: string (毛象内容预览)
  created_at: timestamp
  is_read: boolean
}
```
