## 一、现状与改进方向

### 现有基础

## 三、新增功能

### 3.1 多选转发

**流程**：
1. 长按消息 → 点击[多选] → 进入多选模式
2. 底部出现操作 bar：`[逐条转发]  [合并转发]  [收藏]  [删除]`
3. 顶部显示"已选中 n 条消息"
4. 支持"全选"
5. 点击转发方式 → 选择转发目标

**多选视觉表现**：
- 左侧出现勾选框（✓ / ☐）
- 气泡内容向右移动，留出勾选框空间

**转发对象**：
- 其他聊天对象（邮件笔友等）
- Cha Phone 各 App（Notes、Diary、Photos 等）
- Mastodon 发毛象

**需要实现**：
- [ ] 多选模式的状态管理
- [ ] 左侧勾选框 UI
- [ ] 底部操作 bar
- [ ] 转发目标选择列表
- [ ] 逐条转发和合并转发的区别处理

### 3.2 合并转发卡片

**卡片格式**：
```
📋 KK和Cha的聊天记录
2024-06-24 14:00 - 14:30
━━━━━━━━━━━━━━━━━━━
👤 KK
你怎么想？

👤 Cha
我觉得很有意思...

👤 KK
[图片信息]
━━━━━━━━━━━━━━━━━━━
[点击查看完整聊天记录]
```

**需要实现**：
- [ ] 卡片生成逻辑（最多显示 3 条消息）
- [ ] 点击进入详情视图（显示所有转发的消息）
- [ ] 卡片样式（支持文字、图片、表情、语音等）

### 3.3 收藏系统

**收藏功能**：
- 长按消息 → 点击[收藏] → 消息加入收藏夹

**收藏夹界面**：
- 快速筛选：`[最近使用]  [图片与视频]  [聊天记录]  [语音]`
- 按收藏时间倒序显示
- 最近使用定义：近 3 天有 2 次以上点击打开

**点击收藏卡片**：
- 进入该消息的详情视图

**需要实现**：
- [ ] 收藏状态标记（数据库字段）
- [ ] 收藏夹 UI
- [ ] 访问次数和时间追踪
- [ ] 筛选逻辑

### 3.4 撤回 vs 删除

**撤回**（仅 KK 消息）：
- 前端显示：`你撤回了一条消息\n原文：xxx`
- Cha 能看到"撤回"提示和原文内容
- 数据库保留完整记录

**删除**（KK 和 Cha 都支持）：
- 前端消息消失（下一条气泡上移）
- 数据库保留记录（但标记为已删除）
- 对方知道有消息被删除，但不知道内容

**需要实现**：
- [ ] 撤回标记和原文保存
- [ ] 删除标记（is_deleted）
- [ ] 前端显示差异化
- [ ] 数据库不真正删除，仅标记

## 七、数据模型补充

### Message 表需要的新字段

```
is_deleted: boolean              // 是否已删除
deleted_at: timestamp            // 删除时间

is_recalled: boolean             // 是否已撤回
recalled_at: timestamp           // 撤回时间
recalled_original: string        // 撤回前原文

edited: boolean                  // 是否已编辑
edit_history: array              // 编辑历史
  - version: number
  - content: string
  - edited_at: timestamp

is_favorited: boolean            // 是否已收藏
favorited_at: timestamp          // 收藏时间
favorite_access_count: number    // 打开详情次数
favorite_last_accessed: timestamp // 最后打开时间

relate_regenerated_id: uuid      // 关联的重新生成消息

// 语音相关
audio_url: string                // 音频文件 URL
audio_duration: number           // 时长（秒）
audio_type: "real" | "fake"      // real: Cha 真语音, fake: KK 假语音
audio_transcribed_text: string   // 转文字结果

// 图片相关
image_url: string
image_description: string        // 用户编辑的描述
image_prompt: string             // Cha 生成的提示词
```
