# 引用协议分离验证

## 设计原则：三者分离

1. **引用卡（UI）**：只负责展示，在 DOM 中渲染 `.msg-quote-block`
2. **用户正文（DB content）**：只保存用户真实输入，不混入引用内容
3. **模型上下文（API）**：单独编译引用内容，发送给模型

}

### 3. 模型上下文编译（✅ 正确）

**编译位置**: `app.js:1704-1719`

```js
const compiledMessages = messages.map(msg => {
  if (!msg.replyTo || msg.role !== "user") {
    return { role: msg.role, content: msg.content };  // ← 无引用时使用原始 content
  }

  // 有引用时才编译
  const replyLabel = msg.replyTo.role === "assistant" ? "Cha" : "用户";
  const replyPreview = msg.replyTo.preview || "[消息]";
  const compiledContent = `[引用${replyLabel}的消息]\n${replyPreview}\n\n[用户回复]\n${extractTextFromMessageContent(msg.content)}`;

  return { role: msg.role, content: compiledContent };  // ← 编译后的内容仅用于 API
});

// 发送给模型
return fetch(endpoint, {
  body: JSON.stringify({
    messages: compiledMessages,  // ← 使用编译后的消息
    // ...
  }),
});
```

create table if not exists public.messages (
  id bigint generated by default as identity primary key,
  role text not null check (role in ('user', 'assistant')),

## 数据流验证

### 发送带引用的消息

```
用户输入: "好的"
引用消息: { id: "123", preview: "今天天气不错", role: "assistant" }

1. UI 渲染:
   - 消息气泡显示: "好的"
   - 引用卡显示: "Cha: 今天天气不错"

2. DB 保存:
   content: "好的"  ← 只有用户输入
   reply_to_message_id: "123"
   reply_to_preview: "今天天气不错"
   reply_to_role: "assistant"

3. API 发送:
   content: "[引用Cha的消息]\n今天天气不错\n\n[用户回复]\n好的"  ← 编译后的完整上下文
```

### 加载历史消息

```
从 DB 读取:
{
  id: "456",
  role: "user",
  content: "好的",  ← 原始内容
  reply_to_message_id: "123",
  reply_to_preview: "今天天气不错",
  reply_to_role: "assistant"

1. 重建 replyTo 对象:
   replyTo = { id: "123", preview: "今天天气不错", role: "assistant" }

2. 渲染消息:
   - 消息气泡: "好的"
   - 引用卡: "Cha: 今天天气不错"

3. 后续 API 调用:
   - 从 chatMessages 读取 content: "好的" 和 replyTo
   - 重新编译: "[引用Cha的消息]\n今天天气不错\n\n[用户回复]\n好的"
```
