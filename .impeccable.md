# savePrincessCha — Design Context

## Design Context

### Users
Two specific people (KK and Cha) in a long-distance relationship. KK uses the app to talk with Cha, an AI companion persona. The interface is deeply personal — every interaction carries emotional weight. Used on mobile, often in quiet moments: evenings, before sleep, while listening to music. The app should feel like a sacred private space, not a generic chat app.

### Brand Personality
**Warm. Intimate. Handcrafted.**

- Quiet warmth: Like handwritten letters and worn-soft sweaters
- Personal intimacy: Not polished tech, but something made just for these two people
- Gentle presence: Soft, unhurried, always there

Emotional goals: Safety, tenderness, feeling seen. The interface should disappear into the conversation — never get in the way.

### Aesthetic Direction
**Reference**: KakaoTalk's design completeness + frosted glass/grayscale/hand-drawn mascot aesthetic
**Theme**: Light-primary, clean warm whites, soft grays — not clinical, but lived-in
**Anti-references**: Cold dark developer UIs, generic chat apps (Telegram gray, Discord dark), "AI assistant" aesthetics

Key visual traits:
- Warm off-white backgrounds (`#F7F7F5`, `#F8F8F8`) — never pure white
- Blue user bubbles (`#5B9FF5`) — bright but not harsh
- Light gray incoming bubbles (`#ECECEC`) — paper-like
- Warm accent (`#E8A87C`) — hand-drawn, peach-beige
- Frosted glass on toolbars and nav — translucent, layered depth
- Soft system fonts — no decorative typefaces, legibility is intimacy
- Generous whitespace — the silence between words matters

### Design Principles
1. **Disappear into the conversation** — UI chrome should be invisible when reading; only appear when needed
2. **Every message type feels complete** — text, voice, image, sticker, call cards all have their own language
3. **Warmth over polish** — slight softness preferred over sharp, corporate precision
4. **Mobile-first intimacy** — max 430px, phone-shaped, tactile interactions
5. **Frosted glass as atmosphere** — blur + translucency creates depth without color, reinforcing the quiet, layered feel

### Implementation Notes
- Vanilla JS/CSS/HTML — no build system, no React
- `v2.css` overrides `style.css` — new bubble styles go in `v2.css` or a new `v2-bubbles.css`
- CSS variables defined in `v2.css` `:root` block
- Dark mode via `[data-theme="dark"]` or `@media (prefers-color-scheme: dark)`
- Target: KakaoTalk-level polish across 7 message types
